PROBLEM: PDF Text Extraction Failing (400 Bad Request)
------------------------------------------------------
- Issue: pypdf extracting 0 characters from PDF files
- Root Cause: Some PDFs use different encoding/structure
- Solution: Implemented multi-library fallback extraction chain

Extraction Order (main.py):
  1. pypdf (PyPDF2 replacement) - Primary method
  2. pdfplumber - Secondary fallback
  3. PyMuPDF (fitz) - Third fallback
  4. Raw text extraction - Final fallback

Code Implementation:
  def extract_pdf_text_robust(pdf_bytes):
      # Try pypdf first
      reader = pypdf.PdfReader(BytesIO(pdf_bytes))
      text = "".join(page.extract_text() or "" for page in reader.pages)

      if len(text.strip()) < 100:
          # Try pdfplumber
          with pdfplumber.open(BytesIO(pdf_bytes)) as pdf:
              text = "\n".join(p.extract_text() or "" for p in pdf.pages)

      if len(text.strip()) < 100:
          # Try PyMuPDF
          doc = fitz.open(stream=pdf_bytes, filetype="pdf")
          text = "\n".join(page.get_text() for page in doc)

      return text

PROBLEM: Classification Showing "Freshman" Incorrectly
------------------------------------------------------
- Issue: Parser matching "Freshman" from course names like "Freshman Composition"
- Root Cause: Regex searching for classification anywhere in text
- Solution: Calculate classification from earned credits (most reliable)

Classification Logic:
  if credits >= 90: classification = "Senior"
  elif credits >= 60: classification = "Junior"
  elif credits >= 30: classification = "Sophomore"
  else: classification = "Freshman"

PROBLEM: Credits Showing "100" Instead of Actual Value
------------------------------------------------------
- Issue: Parser matching "100" from "100%" completion percentage
- Root Cause: Greedy regex matching first number
- Solution: Search for specific patterns and exclude false positives

Pattern Matching:
  - Look for "Credits applied: {number}"
  - Look for "Credits Earned: {number}"
  - Exclude value "100" (likely percentage)
  - Validate credit range (1-200)

PROBLEM: Advisor Showing "Prerequisite"
---------------------------------------
- Issue: Parser matching words like "Prerequisite" as advisor name
- Root Cause: No filtering for common false positive words
- Solution: Implemented blocklist of excluded words

Blocklist:
  not_advisor_words = [
      'prerequisite', 'prerequsite', 'required', 'complete', 'incomplete',
      'pending', 'satisfied', 'unsatisfied', 'needed', 'met', 'unmet',
      'information', 'course', 'credit', 'hours', 'gpa', 'grade'
  ]

DATABASE MODEL (models.py)
--------------------------
class SupportTicket(Base):
    __tablename__ = "support_tickets"

    id = Column(Integer, primary_key=True, index=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)

    # Ticket Details
    subject = Column(String(255), nullable=False)
    category = Column(String(50), nullable=False)  # "bug", "feature", "question", "other"
    description = Column(Text, nullable=False)
    attachment_data = Column(Text, nullable=True)  # Base64 encoded
    attachment_name = Column(String(255), nullable=True)

    # Status tracking
    status = Column(String(50), default="open")  # "open", "in_progress", "resolved", "closed"
    priority = Column(String(20), default="normal")  # "low", "normal", "high", "urgent"

    # Admin response
    admin_notes = Column(Text, nullable=True)
    resolved_by = Column(Integer, ForeignKey("users.id"), nullable=True)
    resolved_at = Column(DateTime, nullable=True)

    # Metadata
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
