Same ADK agent, completely new voice pipeline

User speaks...
--- 3-5s silence detection ---
Browser STT: "Where is the CS dept?"
--- POST /chat (sync, wait 2-8s) ---
Full response received (all at once)
--- OpenAI TTS (wait 300ms-1s) ---
Play entire audio clip
--- Done, resume listening ---

5-15 seconds of dead silence between speaking and hearing the answer
UPGRADED (Conversational)
User speaks...
VAD: end-of-speech! (<50ms)
Deepgram STT: Where is the CS "Where is the CS dept?"
SSE /chat/stream: chunks arriving...
Sentence 1 complete -> TTS -> PLAY
...more chunks still generating...
Sentence 2 complete -> TTS -> queue
User interrupts? -> stop TTS, listen

1-2 seconds to first spoken word. User hears answer while rest generates.

Voice Mode State Machine

Clean transitions with barge-in support

State Flow
IDLE
->
LISTENING
VAD active + live waveform
->
PROCESSING
STT + LLM streaming
->
SPEAKING
Sentence-by-sentence TTS
SPEAKING -> LISTENING (barge-in: user speaks)
SPEAKING -> LISTENING (auto: all audio done)
Any state -> IDLE (user clicks stop)

Implementation Phases

Each phase ships independently

1

VAD + Interim Results

Add Silero VAD for instant end-of-speech. Show words appearing in real-time. Biggest UX win, least code.

2

Streaming TTS

Switch from /chat to /chat/stream. Sentence splitter + audio queue. Kills the long silence. User hears first sentence in ~1s.

3

Deepgram STT

Backend WebSocket proxy. Replace browser STT with Deepgram streaming. Faster and more accurate transcription.

4

Barge-In + Waveform

User can interrupt by speaking. Live audio waveform visualization. Full conversational feel.
